reflection retrieval v2 + adversarial benchmark + content conflation prevention

Reflection retrieval subsystem redesign:
- Topic-dense retrieval text generator (_extract_reflection_retrieval_text)
  strips boilerplate, extracts entities/topics, natural language format
- Structured metadata extraction (primary_topic, entities, themes,
  emotional_tone, project_area) stored in ChromaDB
- Reflection-specific query rewriting + dual-query semantic search
- Entity/topic overlap scoring blended with semantic similarity
- Expanded candidate pool (50) + cross-encoder rerank (top 25)
- Metadata-only fallback via ChromaDB where_document
- Improved reflection generation prompt (topic-specific, entity-rich)
- Reflection MRR: 0.39 → 0.93 (+137%)

Adversarial benchmark (v2):
- 868 seed memories (200 targets + 668 near-neighbor distractors)
- 3 nearest-neighbors per target as confusers + 30 random per collection
- Combined MRR: 0.82 → 0.88, R@topK: 0.91 → 0.97
- Per-method reporting, fixed MRR aggregate (retrieval cases only)
- Meta-conversational cases now have retrieval targets (MRR 0.40 → 0.71)
- Facts evaluation analysis documented (0.77 MRR is measurement artifact)

Content conflation prevention (3 composable features):
- Session boundaries in prompt: visual markers when conversations
  cross session boundaries (>2h gap or date change)
- Shared content object tracking: detect lyrics/poems/code/quotes
  via regex, store content_type metadata, retroactive attribution
  detection ("it's by X"), content object rendering in prompt
- Cross-session ambiguity detection: when short user message references
  a phrase appearing in multiple sessions, inject disambiguation note
