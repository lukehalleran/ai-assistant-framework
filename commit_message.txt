feat: add agent session safety layer

Mechanical safeguards for AI coding sessions after a git restore
destroyed uncommitted work. Snapshots repo state before agent work,
blocks destructive git commands, and audits changes afterward.

New components:
- utils/fs_snapshot.py: filesystem manifest (sha256, size, mtime), diffing, CLI
- utils/destructive_op_guard.py: git command classifier (restore/reset --hard/clean/push blocked)
- scripts/agent_session_start.sh: pre-agent snapshot (git state, manifest, filtered untracked tarball, 10-snapshot rotation)
- scripts/agent_session_audit.sh: post-agent audit (manifest diff, branch/HEAD comparison)
- scripts/safe_git.sh: safe git wrapper with env var and one-shot lockfile unlock
- docs/AGENT_SAFETY.md: full workflow documentation and emergency recovery procedures

92 new tests (40 fs_snapshot + 52 destructive_op_guard), all passing.

Also includes: config.yaml model switch to deepseek-v4.
