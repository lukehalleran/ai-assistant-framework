feat: v1.0.0 desktop build — user/dev mode, expanded wizard, installer

First distributable release. Built and tested on Windows 11 with Inno
Setup installer (293MB). Major wizard expansion, user/dev mode system,
and multiple UX fixes for desktop distribution.

User/Dev Mode System (DAEMON_MODE):
- Wizard asks Personal vs Developer on first run
- User mode disables: synthesis pipeline, code proposals, reference
  docs in context, Debug Trace/Logs/Proposals/Synthesis tabs, Memory
  Maintenance UI, system prompt in Provenance
- User mode enables: auto-dedup execution on shutdown (vs dry-run)
- Config gating at end of app_config.py after all flags loaded

Wizard Expansion (gui/wizard.py):
- Flow: WELCOME → INTRO → MODE → API_KEY → TAVILY_KEY → WOLFRAM_KEY →
  (E2B_KEY if dev) → STYLE → NAME → PRONOUNS → OBSIDIAN → WIKI_INDEX →
  BACKGROUND → COMPLETE
- New steps: mode selection, Wolfram Alpha, E2B sandbox (dev only),
  Obsidian vault (double-confirms skip, disables notes if skipped),
  Wikipedia FAISS index path
- Generic write_env_key() replaces duplicated per-key write functions
- Intro message explains response times as a feature ("I think deeply")
- Dev mode intro: "I'm actively evolving" messaging
- Dev mode completion: tips about Proposals, Synthesis, goals.md
- Completion message: "Close and relaunch" (not "refresh the page")

GUI Changes (gui/launch.py):
- Model selector moved from Settings to Chat tab (dropdown in submit row)
- Tabs gated by mode: Debug Trace, Logs, Proposals, Synthesis hidden
  in user mode via Gradio visible= flag
- Memory Maintenance section hidden in user mode
- System prompt hidden from Debug Trace and Provenance in user mode
- Tab overflow menu CSS fix (white-on-white → dark theme)
- Browser tab close triggers graceful shutdown (Gradio unload event)
  for both wizard and main UI when running as frozen exe

Shutdown Improvements (main.py, shutdown_processor.py):
- Daily note generation added to shutdown tasks
- Cross-dedup auto-executes in user mode (CROSS_DEDUP_AUTO_EXECUTE)
- Weekly notes env var override added (was missing)

File Upload Fix (gui/handlers.py, core/prompt/builder.py):
- prepare_prompt now receives merged_input (user text + file content)
  instead of bare user_text — file content was being lost
- Reference docs retrieval suppressed when files uploaded
  (_suppress_reference_docs flag via context.has_files)

Build & Distribution:
- daemon.spec: console=True (closing console/browser tab kills process)
- main.py: stdout redirect to log file as safety net for windowless mode
- Splash screen progress messages during startup
- Inno Setup installer tested (installer/daemon_installer.iss)
- .env.example updated with DAEMON_MODE

Config Changes:
- config.yaml: added daemon.mode (default: user)
- app_config.py: DAEMON_MODE early load, user-mode overrides block,
  CROSS_DEDUP_AUTO_EXECUTE, WEEKLY_NOTES_ENABLED env override

Documentation:
- BUILD_GUIDE.md: full rewrite with installer, mode table, test checklist
- README_INSTALLER.md: updated wizard flow, mode explanation
- PROJECT_SKELETON.md: wizard section rewritten (14 steps)
- ARCHITECTURE_GUIDE.md: user/dev mode section, updated entry points
- QUICK_REFERENCE.md: dedup auto-execute docs
- README.md: wizard summary, installer build command
