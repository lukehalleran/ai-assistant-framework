fix: agentic tool dispatch, reasoning bypass, XML robustness, context pollution

Controller routing: _dispatch_single was missing routes for github,
stackexchange, arxiv, pubmed, hackernews — tool calls returned
round_data=None so rounds never counted, causing infinite loops.

Reasoning bypass: XML-protocol decision phase now uses
_generate_decision_no_reasoning() which skips native reasoning params.
DeepSeek/R1 models were burning the entire 500-token budget on
chain-of-thought, leaving nothing for XML tool markers.

Tool name triggers: messages mentioning "github", "git stats",
"actions", "workflow", etc. now force agentic mode at Tier 1
regardless of message length (previously casual-skipped).

XML robustness: nested tags (<git_stats><query>X</query></git_stats>),
empty tags (<github></github>), and DeepSeek-style
<search_memory><query>X</query><collection>Y</collection></search_memory>
all handled. Empty args default to sensible queries instead of
silently dropping the tool call.

Context pollution: leaked tool XML stripped at block level before
storage. _strip_leaked_xml_blocks replaces _strip_leaked_xml_markers
across all display and storage paths. Bare tool-call lines stripped
from zero-dispatch agentic responses.

Thread extractor: today's date injected into extraction and resolution
prompts so relative dates ("tomorrow", "next Tuesday") resolve to
absolute dates.

Additional: nudge retry for narrated-but-not-executed tool calls,
tool hints in iteration prompts, context-aware casual skip override,
skip_initial_search for tool-triggered queries, gh repo view
positional arg fix.
