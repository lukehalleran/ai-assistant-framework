feat: Add expand_memory agentic tool for temporal context expansion and summary drill-down

New agentic tool that lets the LLM "zoom in" on a memory search hit to see
surrounding conversation turns or, for summaries, the original conversations
that were compressed into it.

## New System

### Memory Expansion (memory/memory_expander.py)
MemoryExpander class with two expansion strategies:
- **timestamp_window**: For conversations, reflections, facts, obsidian_notes —
  fetches chronological neighbors (configurable window, default ±3 turns).
- **source_docs**: For summaries — retrieves the original conversation turns
  that were compressed into the summary, using source_doc_ids metadata or
  temporal_anchor_start/end range. Works retroactively on existing summaries.

Caches results per (memory_id, window, collection) tuple. Session-gated at
EXPAND_MAX_PER_SESSION (default 3). Content truncated per configurable char
limits (anchor: 600, context: 300).

## Agentic Integration
- Tool definition (EXPAND_MEMORY_TOOL_DEFINITION) in types.py with anti-loop
  guidance in description
- SearchDecision fields: wants_memory_expand, expand_memory_id, expand_window,
  expand_collection, expand_reason
- Parsing in protocols.py: NativeToolsHandler (function calling) and
  XMLMarkerHandler (<expand_memory> tag)
- Execution branch in controller.py main loop with ProgressEvent emission,
  session gating, and formatted context accumulation
- GUI thought bubbles: "Recalling..." (conversations) or "Recalling Long Term
  Memory..." (summaries) via new icon mappings in handlers.py

## Supporting Changes
- **get_by_id()** added to MultiCollectionChromaStore for direct doc lookup
- **Thread metadata forwarding**: store_interaction() now propagates thread_id
  and thread_depth from thread detection to ChromaDB conversation metadata
- **Summary backlinks**: shutdown_processor._store_summary() captures source
  conversation doc IDs by querying conversations collection within the block's
  temporal anchor range, stored as comma-separated source_doc_ids metadata
- **Search result format**: _format_memory_results() now shows full doc IDs
  (not truncated) and [collection] tags so the LLM can pass IDs to expand_memory
