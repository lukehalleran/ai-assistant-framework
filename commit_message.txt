feat(actions): registry-driven write actions + GitHub write (issues/comments)

Introduce a single source of truth for human-in-the-loop write actions and add
a new GitHub write action, eliminating the wiring drift that previously made
adding an action painful (edits in 4+ files that were easy to forget).

Write-action registry (single source of truth)
- core/actions/registry.py (NEW): ACTION_SPECS — one ActionSpec per write
  action declaring executor_attr, required/optional params, intent_patterns,
  enable_flag, health_probe_attr, backfill_attr, and instruction text.
  ACTION_SPEC_ORDER gives deterministic iteration. register()/get_spec() API.
  Registered: send_telegram, send_discord, send_email, create_calendar_event,
  github_write.
- All consumers now derive behavior from the registry instead of hand-wiring:
  - executors.py: ActionExecutorRegistry builds its dispatch map from
    ACTION_SPECS (resolving executor_attr to bound methods); adds logging.
  - agentic/protocols.py: registry-driven _detect_action_intent() for the
    XML/no-native-tools fallback; native parse for github_write.
  - agentic/tools.py: registry-driven get_tool_health() via
    _resolve_health_probe(); new _handle_github_write() + DISPATCH_TABLE row.
  - agentic/controller.py: _detect_write_action() rewritten to iterate
    ACTION_SPEC_ORDER (replaces the hand-wired if-ladder); adds
    _backfill_action_params(), _action_instructions(), _backfill_github_write().

New GitHub write action
- core/actions/github_write.py (NEW): create_issue / comment_issue via the `gh`
  CLI (must be installed + `gh auth login`). execute() dispatches by operation;
  github_write_configured() is the tool-health probe. All failures degrade to a
  failed ActionResult — never raise. Write counterpart to the read-only
  core/github_manager.py; only ever runs through the approval flow.
- core/actions/types.py: add ActionType.GITHUB_WRITE; ActionResult.detail dict;
  ActionProposal.risk; PendingActionsStore.get() (non-popping peek).
- core/agentic/types.py: SearchDecision.use_github_write; GITHUB_WRITE tool
  definition added to NATIVE_TOOL_DEFINITIONS (now 21 agentic tools).
- core/orchestrator.py: agentic path handles use_github_write alongside
  use_propose_action; _handle_proposed_action() branches accordingly.

Orchestrator entry point
- core/orchestrator.py: new public process_user_query() (build_context →
  _generate → _persist) so tests/CLI can drive a query without the GUI.

Reliability / robustness
- models/model_manager.py: generate_once() gains a timeout (default from
  tool_timeout_s, 30s) via asyncio.wait_for; returns "" on timeout. Inner
  logic factored into _generate_inner().
- memory/claim_tracker.py: canonicalize_claim() swallows resolver failures;
  extract_claims_from_text() skips short lines, markdown headers/bullets, and
  empty subject/relation after split; cascade_staleness() short-circuits on
  empty doc lists and skips docs missing from the store.
- memory/memory_consolidator.py: skip consolidation on empty summary
  (status=skipped); log extracted claim count.
- memory/memory_retriever.py: null-safe fact scores with a stable
  recency tie-break; _dedup_by_content() now actually dedups on
  whitespace-normalized, lowercased content.
- memory/shutdown_processor.py: register claims then backfill doc_id into the
  reverse index; log when no claims are extracted.

Config
- config/config.yaml: new github_write section (enabled, default_repo,
  operations, max_body_chars, repo_allowlist) + process_query section.
- .gitignore: ignore .gh_token and commit_message.txt.

Tests
- tests/unit/test_github_write.py (NEW): create/comment dispatch, gh-unavailable
  degradation, unknown-operation/missing-repo guards.
- tests/unit/test_tool_wiring_parity.py (NEW): every ActionType has a spec +
  executor; ACTION_SPEC_ORDER covers all specs; every SearchDecision dispatch
  flag exists; dispatch handlers unique; github_write spec shape.
- tests/unit/test_process_user_query.py (NEW): happy path + str fallback.
- tests/test_memory_consolidator.py: empty-summary skip + claim-count cases.
- tests/unit/test_claim_tracker.py: resolver-failure safety, header/bullet
  skipping, empty-after-split, and min-length cases.
