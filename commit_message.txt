fix: visual memory entity resolution, profile fact staleness & categorization cleanup

Visual memory retrieval replaced naive substring matching with proper entity
resolution via extract_graph_entities(), fixing wrong-entity photo pulls (e.g.
asking about Paczki but getting Flapjack). Added junk entity filtering for
caption artifacts (colors, quantifiers, hedges), visual-intent proximity
disambiguation for multi-entity queries, and target_entities hard-filtering
in VisualRetriever with direct metadata fallback when CLIP can't rank minority
entities above the similarity threshold. Short messages (<=5 words) without
intent override now suppress visual retrieval entirely. Hardened [VISUAL
MEMORIES] prompt instruction to prevent LLM from commenting on irrelevant
images.

meeting_with_), suffixes (_appointment, _meeting, _intake, _consumption), and
exact matches (meal, drank_alcohol) -- fixes stale event facts (career center,
job fair) leaking into prompt months after they occurred.

categorize_relation(), added ~40 direct lookup entries, ~12 prefix mappings,
~30 category tokens. Ran cleanup: 654 facts recategorized, 29 merged, 51
expired. Preferences category reduced from 428 to 54 current facts (87%).

intent classifier ("do you see X", "show me X"), removed overly broad
"update"/"updates" from web search NEWS_KEYWORDS, added visual query
suppression patterns. Memory search now takes priority over web search when
LLM detects recall intent. Agentic context inventory expanded with 5 missing
sections (visual memories, git commits, knowledge graph, threads, insights)
