feat: document generation pipeline + daemon self-notes system

Two new knowledge subsystems that turn retrieval + memory + agentic tooling
into durable artifacts.

Document Generation (knowledge/document_generator.py)
- User says "write a report about X" → gathers sources from web search,
  ChromaDB (wiki, notes), generates outline + draft with inline citations
- Output: documents/reports/ or documents/summaries/ with YAML frontmatter,
  [WEB_N]/[WIKI_N]/[NOTE_N] inline citations, Sources section
- documents/index.json tracks all generated docs
- Direct invocation from handler (bypasses agentic loop for reliability)
- Also registered as agentic tool generate_document for ReAct loop use
- LLM topic refinement strips conversational filler from queries
- Token budget doubled to compensate for reasoning token consumption

Daemon Self-Notes (knowledge/daemon_notes_manager.py)
- Daemon leaves structured notes for its own future sessions:
  implementation decisions, architecture rationale, risks, next steps
- New ChromaDB collection: daemon_self_notes (ground_truth=False always)
- Two creation paths: user-directed ("save a note for yourself about X")
  and autonomous (create_daemon_note tool in ReAct loop)
- Retrieved via get_daemon_self_notes() in context gatherer, max 2 per prompt
- Prompt section [DAEMON SELF-NOTES] with caveat: "not user-authored facts"
- Scoring: COLLECTION_BOOSTS['daemon_self_notes'] = -0.05 (below user content)
- Registered in eval section_registry (assembly_order=28, ablatable)
- Added to proactive_context bundle for ablation testing

Infrastructure changes:
