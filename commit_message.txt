feat: Integrate Obsidian vault into RAG pipeline with hybrid retrieval

Adds personal notes from user's Obsidian vault (~Documents/Luke Notes) to the
context retrieval system. Notes are embedded in a dedicated ChromaDB collection
and retrieved using hybrid search (1/3 keyword + 2/3 semantic).

## New Files

- `knowledge/obsidian_manager.py` - Vault parsing, embedding, and retrieval
  - Smart chunking: whole notes <1500 chars, else split by ## headers
  - Extracts #tags and [[wiki links]] as metadata
  - Strips YAML frontmatter
  - Hybrid retrieval: keyword match on title/tags + semantic similarity

## Modified Files

- `config/app_config.py` - Added OBSIDIAN_* configuration constants
- `memory/storage/multi_collection_chroma_store.py` - Registered obsidian_notes collection
- `main.py` - Added CLI commands: embed-vault, vault-stats, clear-vault
- `core/prompt/context_gatherer.py` - Added get_personal_notes() method with citation tracking
- `core/prompt/builder.py` - Added parallel retrieval task, gating, [USER'S PERSONAL NOTES] section
- `core/prompt/token_manager.py` - Added personal_notes to PRIORITY_ORDER (priority 6)
- `gui/launch.py` - Fixed Debug Trace tab CSS (white text on white background)
- `docs/PROJECT_SKELETON.md` - Documented obsidian_manager.py module

## Usage

```bash
# Index vault (one-time)
python main.py embed-vault

# Force full re-index
python main.py embed-vault --force

# Check status
python main.py vault-stats

# Clear collection
python main.py clear-vault
```

Notes automatically appear in prompts when relevant to user queries, filtered
through the existing 3-stage gate system (Cosine → Blended → CrossEncoder).
