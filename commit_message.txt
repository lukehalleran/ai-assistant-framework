## Smarter Query Expansion (NEW)
- rank_expansion_candidates() in graph_utils.py — ranks expansion terms by lateral connectivity (non-hub edge count) instead of name length
- Score = min(non_hub_edges * 0.3, 1.0) + single-word bonus (+0.1) / 3+ word penalty (-0.1)
- _is_expansion_junk() filters: empty/≤2 chars, 4+ words, digit-starting, temporal, measurement, verb phrases
- context_gatherer.py:_expand_query_with_graph() now calls rank_expansion_candidates() instead of get_related_display_names() + name-length sort

## Junk Node Prevention at Ingestion (NEW)
- Tightened _is_graph_worthy_object() in memory_storage.py with 3 new rejection filters:
  - Temporal/duration: "2 years", "6 months", "once a week"
  - Measurements: "5'11\"", "20lbs", "10000iu", "500mg"
  - Verb phrases: "stopped being religious", "finishing grad school", "working remotely"
- Rejected objects stored as subject-node metadata instead of creating graph nodes
