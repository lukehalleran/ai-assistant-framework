docs: Synthesis calibration plan, wiki subset builder, cross-doc audit fixes

New files:
- docs/SYNTHESIS_CALIBRATION_PLAN.md — step-by-step plan for testing and
  calibrating the synthesis pipeline using curated FAISS wiki subset and
  generated data. Covers both synthesis pathways (shutdown dreaming +
  proactive surfacing), fact-to-graph ingestion flow, 5-phase test plan
  with concrete success criteria, and threshold tuning guide.
- scripts/build_wiki_subset.py — builds a domain-focused FAISS index from
  the full Wikipedia corpus. 10 domain categories (health_medical,
  education_statistics, computer_science, philosophy_mind, history_politics,
  geography_places, hobbies_interests, psychology_relationships,
  cross_domain_science, verification_pairs) with seed titles + keyword
  patterns. Two input modes: stream from tar.zst or filter from full build.
  Supports --dry-run, --max-articles, adaptive IVF parameters for smaller
  indices. Output: wiki_data_subset/ with FAISS index, metadata parquet,
  embeddings mmap, index_meta.json with matched titles.

Modified scripts:
- scripts/build_faiss_index.py — major rewrite adding streaming tar.zst
  support (no full extraction needed), 2-phase pipeline (extract → build
  FAISS, runnable separately via --phase), resume checkpointing with
  _checkpoint.json for crash recovery, memory-mapped embeddings via numpy
  mmap, chunked metadata parquet parts merged at end, argparse CLI with
  --tar/--phase/--parquet-dir options, adaptive IVF centroid count
  (4*sqrt(n)), index_meta.json output summary. Legacy extracted-parquet
  mode preserved as fallback.
