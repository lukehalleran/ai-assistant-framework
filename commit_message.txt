feat: Google OAuth2 + Calendar integration, Gmail API email, ephemeral fact filtering

Google OAuth2 (core/actions/google_auth.py): installed-app flow with
3 scopes (gmail.send, calendar.readonly, calendar.events). Token
persistence at data/google_token.json, auto-refresh, scope-upgrade
detection, has_scope() gate. Singleton via get_google_auth().

Google Calendar read (core/actions/google_calendar.py): fetch_upcoming_events()
with 5-min module-level cache, httpx GET to Calendar API, returns
minimal fields (summary, start, end, all_day, location). Injected
into prompt as [GOOGLE CALENDAR] section via gatherer_knowledge.py
get_google_calendar_events() + builder.py parallel task.

Google Calendar write (core/actions/google_calendar_create.py):
create_calendar_event() validates scope, POSTs to Calendar API.
Executor wired (was stub). Protocol fix: protocols.py now detects
calendar_create_event, accepts summary instead of message, forwards
all 7 calendar params (summary, description, start_time, end_time,
time_zone, calendar_id, location). Text parser no longer blindly
prepends send_ to non-messaging action types. Tool health in
tools.py now lists calendar_create_event when enabled. types.py
propose_action schema extended with calendar fields, required
relaxed to ["action_type", "reason"].

Gmail API email (core/actions/email.py): rewritten — Gmail API
primary path via google_auth (httpx POST to gmail/v1/users/me/
messages/send with base64 RFC 5322). SMTP fallback only if Gmail
unconfigured/unauthenticated. No SMTP fallback after Gmail attempt
(prevents duplicate sends).

Ephemeral fact filtering: dual-layer blocking in fact_extractor.py
and llm_fact_extractor.py — blocks PROFILE_EPHEMERAL_RELATIONS
predicates and boolean-only values at extraction time. LLM prompt
updated to explicitly discourage transient state. memory_retriever.py
replaces hardcoded _EPHEMERAL_PREDICATES with config-driven
_get_ephemeral_predicates(), adds TTL filter (PROFILE_EPHEMERAL_TTL_HOURS
default 24h) to drop stale ephemeral facts from retrieval.

Backup: scripts/backup_data.sh uploads encrypted backup to Backblaze
B2 via rclone after local backup, prunes remote backups older than
KEEP_DAYS.

Prompt changes: new [UPCOMING SCHEDULE] and [GOOGLE CALENDAR]
sections in formatter.py. Section registry now 31 entries (was 29).
system_prompt.txt updated with section documentation.

Config: InternetActionsSection extended with google_client_id,
google_client_secret, google_token_path, google_calendar_enabled,
google_calendar_max_events, google_calendar_lookahead_days. YAML
section updated. app_config.py adds GOOGLE_CALENDAR_* and
INTERNET_ACTIONS_GOOGLE_* constants with env var overrides.

New files: core/actions/google_auth.py, google_calendar.py,
google_calendar_create.py, tests/unit/test_calendar_create.py,
test_calendar_prompt.py, test_gmail_send.py, test_google_auth.py,
test_google_calendar.py
