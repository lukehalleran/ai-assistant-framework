real-data benchmark + fact ranking fix + reflection embedding cleanup

Add real-data retrieval benchmark (64 cases sampled from production
ChromaDB) alongside existing synthetic benchmark. Fix two
collection-specific ranking bugs exposed by real-data testing.

Fact ranking: get_facts() was re-sorting ChromaDB results by
0.7*confidence + 0.3*recency, destroying semantic ordering. New
formula: 0.60*semantic + 0.20*confidence + 0.20*recency with a
semantic floor (relevance < 0.30 caps recency at 0.05). This alone
fixed 17 of 24 real-data ranking failures.

Reflection embedding: all reflections started with identical
boilerplate ("### What went well - The assistant effectively...")
causing BGE-small to map them to nearly the same vector.
_clean_reflection_for_embedding() strips 16+ boilerplate patterns
before embedding, preserving original markdown in metadata. Fixed
remaining 7 of 24 failures.

Benchmark harness: content-hash identity matching replaces fragile
prefix/marker matching. Per-collection retrieval routing via
retrieval_method field. Real-data sampler script for reproducible
benchmark generation from production data.
