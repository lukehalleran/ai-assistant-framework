feat: autonomous self-note guardrails — session cap, dedup, failure gating

Hardens the autonomous note-writing path (create_daemon_note agentic tool)
to prevent memory pollution, spam, and feedback-loop drift.

Guardrails added to daemon_notes_manager.py:
- Per-session cap: max 3 autonomous notes per session, resets on new session_id
- Semantic dedup: queries daemon_self_notes before writing, skips if similarity >= 0.85
- Session ID tracking: session_id field added to DaemonNote dataclass + ChromaDB metadata
- create_autonomous_note() method: wraps create_note() with all guardrails,
  returns None (silent skip) when rejected

Status enum expanded:
- Added tentative, hypothesis, failed_attempt to VALID_STATUSES
- Autonomous notes default to status=tentative, confidence=tentative
- Allows marking failed approaches distinctly from confirmed decisions

Agentic tool dispatch (core/agentic/tools.py):
- _execute_create_daemon_note now uses create_autonomous_note() instead of create_note()
- Singleton _daemon_notes_manager for session cap tracking across agentic rounds
- Returns "[Self-note skipped by guardrails]" when cap/dedup rejects

Tests: 9 new guardrail tests (48 total for daemon_notes_manager)

- Status defaults to tentative for autonomous notes
