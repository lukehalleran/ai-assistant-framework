feat: Google Contacts + Gmail search contact resolution, email-by-name support

Contact resolution system enabling "email Maegan" without requiring
a full email address. Three-tier lookup: saved Google Contacts →
Other Contacts → Gmail header search fallback.

Google Contacts (core/actions/google_contacts.py): People API
integration with search_contacts() (saved), search_other_contacts()
(auto/other), resolve_contact() (combined 3-tier with Gmail fallback).
5-min module-level cache keyed by (endpoint, normalized_query).
Lazy warmup request per People API recommendation. Skips contacts
without email addresses. Returns minimal fields: name, email, source.

Gmail header search (core/actions/gmail_search.py): Searches Gmail
messages matching a name query, fetches From/To metadata headers only
(never reads message bodies). Concurrency-limited to 5 parallel
message fetches via asyncio.Semaphore. Deduplicates by email, filters
out user's own address. Requires gmail.readonly scope.

OAuth2 scopes: Added contacts.readonly, contacts.other.readonly,
gmail.readonly to SCOPES list (now 6 total). Existing
needs_scope_upgrade() detects mismatch and prompts re-auth.

Recipient resolution — two layers:
1. Proposal-time (core/agentic/tools.py): _resolve_email_recipient()
   in _dispatch_action_proposal() resolves name→email before storing
   proposal, so confirmation card shows resolved address.
2. Execution-time (core/actions/email.py): _resolve_recipient()
   fallback in send_email() for non-agentic paths.

Agentic gate (core/agentic/gate.py): Broadened email-by-name
patterns — now catches "send [name] an email", "email" + "send"/
"draft"/"compose" co-occurrence, and "email [name]" at start of
message. Triggers agentic routing for contact resolution.

Protocol parsing (core/agentic/protocols.py): Added <propose_action>
and <lookup_contact> XML marker patterns for XMLMarkerHandler. Added
<invoke> fallback parser for Anthropic-style function call XML. Tool
name aliases: search_contacts, find_contact, search_gmail,
search_email, gmail_search, search_inbox — all route to
wants_lookup_contact. Both NativeToolsHandler and XMLMarkerHandler
handle these.

XML stripping (gui/handlers.py, gui/launch.py): 3-layer defense
strips <function_calls>, <invoke>, <propose_action>, <lookup_contact>
blocks from response text. Handles both closed and unclosed tags.
Safety-net strip in launch.py before final UI render.

Inline contact resolution (gui/handlers.py): When models emit
<invoke name="lookup_contact/search_email"> as text (both agentic
and enhanced paths), handler resolves contact inline via
resolve_contact(), auto-creates send_email ActionProposal if email
intent detected. _find_email_draft() searches chat_history for
structured draft content; returns None if no draft found to prevent
sending meta-commentary as email body.

Agentic tools (core/agentic/tools.py): _dispatch_lookup_contact
and _execute_lookup_contact for search-round dispatch. Config guard
checks all three enable flags. Tool health reports lookup_contact
status (removed from tool offerings to prevent model confusion, but
dispatch handles it if models call it from context).

XML prompt (core/agentic/types.py): Added propose_action as tool #14
in AGENTIC_SYSTEM_PROMPT_INJECTION with auto-resolution guidance.
Tool selection guidelines table updated. Strong emphasis: "for email,
just use propose_action with the name — no lookup needed."

Config: google_contacts_enabled (default true), google_other_contacts_enabled
(default true), google_gmail_search_enabled (default true) in schema.py,
app_config.py, config.yaml. All support env var overrides.

New files: core/actions/google_contacts.py, core/actions/gmail_search.py,
tests/unit/test_google_contacts.py

Tests: 27 new in test_google_contacts.py (config gating, API parsing,
dedup, cache, error handling). 5 new in test_gmail_send.py
(TestRecipientResolution). All 72 related tests passing.

Docs: updated CLAUDE.md, QUICK_REFERENCE.md, PROJECT_SKELETON.md,
ARCHITECTURE_GUIDE.md, AGENTIC_SEARCH.md, FORMAL_MODEL.md, GOALS.md.
Module contracts updated in all changed .py files.
