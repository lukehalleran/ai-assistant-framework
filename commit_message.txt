  milestones, contributors, code search

- Config: github_api section (enabled, timeout_s, max_output_lines, repo)

ChromaDB lazy collection initialization:
- Collections registered as None placeholders in __init__(), lazily
  opened on first access via _get_collection(name)
- Eliminates eager initialization of all 13 collections at startup
- _initialize_collections() → _initialize_all_collections() (stats only)
- All internal usages updated: self.collections[name] → self._get_collection(name)
- New get_ids_by_timestamp_range(collection, start, end) method for
  efficient timestamp-filtered queries (replaces list_all scan in shutdown)

Startup parallelization:
- MemoryCoordinator: graph+resolver, user profile, and claim index
  loaded in parallel via ThreadPoolExecutor(max_workers=3)
- main.py: SentenceTransformer model loads serialized via threading lock
  (PyTorch Module.to() is not thread-safe for concurrent model loads)
- Removed eager embedder preload (_get_embedder("all-MiniLM-L6-v2"))
- User profile shared from coordinator → orchestrator (avoids duplicate
  disk read)

Shutdown parallelization:
- Shutdown phases restructured into parallel groups via asyncio.gather:
  Phase A (parallel): facts + LLM facts + behavioral patterns + skills
  Phase B (parallel): proposals + impl tracking + threads + synthesis + wiki
  Phase C (sequential): graph save, category cache, cross-dedup
- Source doc ID lookup uses get_ids_by_timestamp_range() instead of
  list_all() full-scan
