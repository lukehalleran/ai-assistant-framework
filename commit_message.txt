feat: standalone grab-and-go utilities — agent_guard + project_proposer

Two self-contained single-file utilities extracted from Daemon, zero coupling
to the main project. Each file is drop-in ready for any Python 3.10+ project.

agent_guard.py (952 lines, stdlib only)
- Three-layer agent safety: git command classifier, shell command classifier,
  Python FS monkey-patching (10 functions: os.remove, shutil.rmtree, etc.)
- ContextVar-based agent_mode() — async-safe, only guards agent-originated calls
- Two-tier blocking: always-blocked (., .., /, ~, *) + configurable protected paths
- Unlock via env var (ALLOW_DESTRUCTIVE_OPS=1) or one-shot lockfile

project_proposer.py (685 lines, stdlib only, optional openai/anthropic SDK)
- Scans project docs (README, CLAUDE.md, GOALS.md, etc.), git log, structure,
  and metadata (pyproject.toml, package.json, Cargo.toml, go.mod)
- Generates structured feature proposals via pluggable LLM callable
- Built-in adapters for OpenAI and Anthropic, or bring-your-own callable
- CLI: python project_proposer.py --provider openai --model gpt-4o

Tests: 213 total (154 guard + 59 proposer), 0.36s, no API keys needed
Docs: Architecture + skeleton docs for both, module contracts on all files
Verified against Flask (Python), Ruff (Rust), Express (Node) repos
