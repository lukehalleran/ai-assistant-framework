feat: Document retrieval pipeline — table extraction, full-doc tool, batch embedding

Uploaded PDFs and DOCX files with tables (schedules, grade rubrics, etc.)
were silently losing tabular data. Now tables are extracted as markdown
pipe tables, a new agentic tool retrieves complete documents by title,
and bulk embedding uses batched ChromaDB writes.

--- MODIFIED FILES ---

utils/file_processor.py
  - PDF: new _extract_pdf_with_tables() using pdfplumber.extract_tables()
    renders tables as markdown; text outside tables extracted via outside_bbox()
  - DOCX: new _extract_docx_with_tables() using python-docx walks document
    body in order, renders tables as markdown (replaces docx2txt which dropped
    table content entirely)
  - Module contract updated with new methods and dependencies

utils/text_chunking.py
  - New chunk_by_size() function: fixed-size paragraph-boundary chunking with
    overlap, used as fallback for large documents with no ## headers
  - chunk_by_headers() now falls back to chunk_by_size() when content exceeds
    threshold but has no ## headers (e.g. plain-text PDFs)
  - Module contract rewritten

knowledge/reference_docs_manager.py
  - New get_full_document(title): fetches all chunks by title, reassembles in
    chunk_index order. Fuzzy title matching via _fuzzy_resolve_title() when
    exact match fails (case-insensitive word overlap scoring)
  - New list_document_titles(): sorted distinct titles for discoverability
  - upload_document() and upload_text() now use add_batch_to_collection()
    for single embedding pass + disk write per document
  - DEFAULT_MAX_DOCS_PROMPT: 5 → 15
  - Module contract updated
