feat: add fetch_url agentic tool, fix shutdown double-run, harden thinking detection

New agentic tool: fetch_url — reads web page content directly by URL via
Tavily extract API (up to 10K chars). Solves the gap where web_search finds
pages but can't read them. Full wiring: types.py (SearchDecision fields,
FETCH_URL_TOOL_DEFINITION, XML prompt instructions), protocols.py (native
tool parsing, XML marker pattern, factory param), tools.py (dispatch +
execute via _tavily_extract, URL reroute from web_search, citation
registration), formatters.py (format_fetch_url_context), controller.py
(_dispatch_single routing, Round 1 auto-fetch for URLs in user messages).

Agentic gate improvements:
- Tier 1 web search keywords ("web search", "search for", "search online",
  etc.) bypass LLM trigger for instant agentic activation
- URL detection (http/https) in user messages triggers agentic mode and
  auto-fetches as Round 1 instead of web search
- Explicit search keywords bypass META_CONVERSATIONAL/CASUAL_SOCIAL intent
  veto so mixed casual+search messages work
- Skip initial search when Tier 1 fires without LLM-generated terms, letting
  the agentic loop decide (fetch_url vs web_search)

XML marker alias patterns: <web_search>query</web_search>,
<web_search query="...">, <search_memory query="..."> now parsed correctly
by XMLMarkerHandler instead of leaking as raw text. Covers attribute-style
hallucinated formats from models using native tool names as XML tags.

Thinking leak defense:
- Sentence-level patterns (_THINKING_SENTENCE_PATTERNS) catch single-paragraph
  thinking dumps that line-anchored patterns miss ("The user wants me to...",
  "Let me fire...", "in the previous attempt...")
- _count_sentence_pattern_hits() scans without line boundaries
- _detect_untagged_thinking() handles 1-2 line text via sentence patterns
- likely_untagged_thinking() tries sentence patterns before line patterns
- Agentic path: final output now runs untagged thinking detection + stripping
- Stuck thinking recovery: if streaming ends while showing "Thinking...",
  parses accumulated output and yields whatever answer text exists
- Enhanced path: _strip_leaked_xml_markers() applied to display and storage

Shutdown fix: _shutdown_requested flag checked in main.py finally block to
prevent signal handler's shutdown tasks from running twice (was causing
OSError EROFS when filesystem went read-only during duplicate LLM calls).

Bug fixes:
- controller.py _dispatch_single was missing wants_fetch_url and
  wants_recall_image routing (LLM emitted fetch_url 7x but silently dropped)
- controller.py _is_low_quality_result and _generate_relaxation_suggestion
  restored (deleted during refactor but still referenced)
- handlers.py _re UnboundLocalError: local `import re as _re` at line 887
  shadowed module-level import, crashing agentic URL extraction every time
- System prompt updated: Daemon can now fetch URLs and see images

76 new tests in tests/unit/test_fetch_url.py covering SearchDecision fields,
tool definition, XML/native parsing, alias patterns, dispatch routing, URL
reroute, execute with mocked Tavily, formatter, XML marker stripping,
sentence-level thinking detection, and handler URL detection.
