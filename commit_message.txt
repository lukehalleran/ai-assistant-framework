fix: gate visual memory by model vision support and query intent

Problem: non-vision models (DeepSeek, GLM) received image data in API
calls, causing 404 "No endpoints found that support image input". Also,
casual greetings like "hey brother" triggered unnecessary CLIP image
retrieval.

Model gating:
- Added ModelManager.supports_vision(model_name) — True for GPT-4o+,
  Claude, Gemini; False for DeepSeek, GLM, local models
- generate_async() silently drops images when active model lacks vision
  support, preventing API errors

Intent gating:
- CASUAL_SOCIAL, EMOTIONAL_SUPPORT, META_CONVERSATIONAL intent profiles
  now set max_visual_memories=0 — skips CLIP retrieval entirely for
  greetings, emotional queries, and meta-conversation

Docs updated: CLAUDE.md, ARCHITECTURE_GUIDE, PROJECT_SKELETON,
PROMPT_BUILDING_PIPELINE, FORMAL_MODEL
