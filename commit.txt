## core/response_planner.py (new)

ResponsePlanner — lightweight pre-answer planning and post-answer review.

- ResponsePlan (Pydantic): key_points, tone, avoid, strategy
- ReviewResult (Pydantic): passes, confidence, issues, suggestion
- create_plan() — LLM call (~200 tokens) produces response plan from
  query + context signals (intent, tone, topics, thread depth)
- review_answer() — LLM call (~300 tokens) checks response against plan
- should_plan() — bypass for small-talk, crisis/elevated tone, disabled config
- format_plan_injection() — formats plan as system prompt section

Both calls are advisory — failures return None and never block response.

## Orchestrator integration (core/orchestrator.py)

- __init__: creates ResponsePlanner when RESPONSE_PLANNING_ENABLED
- build_full_prompt(): runs planner in parallel with
  build_prompt_from_context() via asyncio.wait(). Plan injected into
  system prompt before _assemble_prompt(). Plan stored on instance
  (_current_response_plan) for review gate in handlers.py.

## Review gate (gui/handlers.py)

Post-answer review gate added after uncertainty fallback block:
- Runs after streaming completes, after uncertainty detection
- If plan exists AND review fails with confidence >= 0.80, retries via
  agentic search with review feedback (same pattern as uncertainty retry)
- Skipped if uncertainty already retried (_uncertainty_retry_done)
- Non-blocking on failure — any exception skips the gate silently
