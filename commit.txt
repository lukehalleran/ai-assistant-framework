## Prompt builder: preserve partial context on retrieval timeout

core/prompt/builder.py — replaced asyncio.wait_for(asyncio.gather(...))
with asyncio.wait(...) so tasks that finish before the 30s deadline keep
their results. Previously a global timeout wiped everything.

- done tasks: result extracted via task.result(), exceptions default to []
- pending tasks: cancelled, fall back to []
- single warning names pending sections; per-task failures stay at warning
- normal diagnostic logs demoted from warning → debug (15 calls):
  IMAGE DEBUG, CONTEXT GATHERING, DEBUG RECENT, BEFORE/AFTER HYGIENE_AND_CAPS,
  MEMORY TOP-UP, START OF SUMMARIES BLOCK, BEFORE/AFTER get_summaries,
  BEFORE/AFTER TOKEN BUDGET, _assemble_prompt DEBUG RECENT block,
  "No duplicate sections" check

## python -m pytest path fix

conftest.py (root) — inserts project root into sys.path so system Python can
  resolve internal modules without requiring the venv
config/__init__.py, gui/__init__.py, knowledge/__init__.py, memory/__init__.py,
models/__init__.py, processing/__init__.py, utils/__init__.py — empty package
  markers so project directories take priority over any system packages with
  conflicting names (e.g. system 'utils' package)

Result: both `python -m pytest tests/unit/ -q` and `venv/bin/pytest` pass
  with 2007 tests, 0 failures (was: 5 failures with venv, 112 collection
  errors with system python)
